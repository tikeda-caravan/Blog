AWS Well-Architectedフレームワークとは?

  • AWSは完全にセルフサービスであり、開発者にとって制約が少なく、自由かつ簡単にサービスを組み合わせて開発できるのが特徴

  • 業務サービスの本番稼働を目的として利用されるようになると、開発者側の自由な設計が災いして、AWSサービスに障害等が発生して本番サービスに影響が出たり、性能・コスト・セキュリティ・運用の面で、自由な設計に起因して課題も出てくるようになりました

  • そこで、AWS固有の設計ノウハウを一般化して普及させる機運が高まり、さまざまな業界のAWSのソリューションアーキテクト(AWS上でのシステム構成とAWS内部構造の両面を把握している技術者)が、自らのノウハウやベストプラクティスをリファレンスとして公開しました。それを標準化したものである。

  • AWS Well-Architectedフレームワークは「オペレーショナルエクセレンス」「セキュリティ」「信頼性」「パフォーマンス効率」「コスト最適化」の5つで構成されている。

AWS認定ソリューションアーキテクト - アソシエイトで最も多く出題される範囲が

  • 高い回復性
  • パフォーマンス
  • セキュアアプリケーション

の3つである。

  • 高い回復性は、Multi-AZを中心にしたSLA1を意識しつつ、ELBやRDSを中心にWeb層とDB層の可用性を以下に保つかを問われる。
  • AWSならではの豊富なコンピューティングリソースを活かして、どのようにスケールアウトやスケールアップをさせるかが問われる。
  • AWSのようなパブリッククラウドならではの責任分界点や認証などを第9章のIAM(Identity and Access Management)を中心に活用し、セキュアアプリケーションをどのように構成するかが問われる。

1.4 AWSを理解するには

S3とEC2から理解する

  • S3を基盤にしたサービスは、SQS・SNS・DynamoDBなどである。
  • EC2を基盤にしたサービスは、RDSやELBなどである。

上記2つがほぼ基盤なので、そこから理解することが望ましい

メモ: IPアドレッシングとはネットワーク上の機器にIPアドレスを割り振ることです。

VPCを理解する

  • VPCはSDN(Software Defined Network)2であり、慣れていないと少し戶惑います。

Amazon Virtual Private Cloud (Amazon VPC) は、定義した論理的に分離された仮想ネットワークで AWS リソースを起動できるようにするサービスです。独自の IP アドレスの範囲の選択、サブネットの作成、ルートテーブルやネットワークゲートウェイの設定など、仮想ネットワーク環境を完全に制御できます。仮想プライベートクラウド内のほとんどのリソースに IPv4 と IPv6 の両方を使用できるため、リソースとアプリケーションへの安全で簡単なアクセスが保証されます。

リソース指向でIAM と CloudFormation を理解する

  • リソースとは、EC2の仮想インスタンスなどのクラウドサービスの対象のこと

  • リソース指向とは、この対象を中心にポリシー制御を考えることを指す。

    ポリシー制御とは、アプリケーションの適切な使用とパフォーマンスの確保、お客様の環境にに対する脅威を阻止するための次世代セキュリティ プラットフォームのアクションを表しています。

  • IAMユーザーの考え方は一般的で、IAMユーザーにアクセス制御を加えること

  • EC2であればIAMロールでアクセス制御を加える

  • S3であればバケットポリシーでアクセス制御を加える

  • CloudFormationは、リソース情報をテンプレートとしてJSON形式でまとめたものである。

Route 53 を使って DNS を理解する

  • AWSのサービスには、IPアドレスではなくAmazonが提供するFQDN(Fully Qualified Domain Name)でアクセスする、という特徴があります。
  • WebアクセスにFQDNが用いられているのと同様に、スケールさせたりアプリケーションリリースを切り替えたりするのに、DNS3が有効な手法だから
  • AWSの環境に移行すると、DNSをベースにしたアーキテクチャを考慮する必要があり、AWSが提供しているDNSサービスであるRoute 53を活用していくことになります。 > Route 53は、パブリックIPだけでなくVPC内のプライベートIPにも対応しています。また、IPv4とFQDNをマッピングするAレコードだけでなく、別名を付与するCNAMEレコード、IPv6に対応するためのAAAAレコード、ヘルスチェックをベースにした負荷分散やフェイルオーバーにも非常に有用です。
